# WindSentinel Versioning Policy

## Baseline
- first release line starts at `v1.0`

## Minor version rule
- if the overall architecture is not modified, every 2 newly accepted requirements increments one minor version
- example: `v1.0` -> `v1.1`

## Tagging intent
- every releasable version must be represented in git and tagged
- release notes should accompany each releasable version

## Notes
- this policy was provided by the user and should only change with explicit user confirmation
