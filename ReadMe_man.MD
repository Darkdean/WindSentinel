帮我用rust语言编写一个针对macos的安全软件，客户端命名为WindSentinelAgent，客户端功能要求为：
1、可监控包括系统进程在内的所有进程行为以及监控的系统时间点，包括进程pid，进程action，进程hash，操作的文件/网络/数据库等，进程参数，父进程hash，父进程pid，父进程参数等。
2、对网络操作，需要监控sip，sport，dip，dport，协议，如果协议是http/https/gopher/等，需要记录具体的访问url以及访问方式，如果访问方式是post/put，则还需要记录附加的数据内容
3、对记录的数据暂时本地保存到/tmp目录下的log.dat文件，其格式为记录数据加密并压缩后的格式
4、每间隔5秒，将log.dat数据的增量进行推送，并且监控log.dat的数据量大小，如果超过500M，则将增量数据推送完成后对log.dat文件清空
5、客户端需要有监视自身健康状态的能力与上报能力，健康状态包括，客户端权限是否正常，客户端各进程是否正常，客户端所在机器的硬件支持信息，客户端运行时内存、CPU、网络、硬盘等资源使用等情况，加密后定时5秒上报到服务器一次。
6、客户端每隔5秒，会向服务器获取一次策略下发请求信息，用于定期修正针对客户端的行为，例如针对某行恶意行为进行消灭进程，阻断进程网络使用，阻断进程操作文件/目录功能等
7、客户端需要额外功能为：
  7.1、远程shell连接，该shell具有对macos文件系统以及用户目录文件空间的所有权。远程shell连接功能仅仅在服务器下发指令后，由客户端获取策略后，开启加密的反向shell进程，连接至服务器，并在服务器端操作。
  7.2、远程锁功能，该功能是针对mac电脑遗失后，防止数据丢失设计，当策略获取到锁定功能时，会自动将用户目录完整加密，加密时，使用RSA算法，加密的公私钥由服务器存放，客户端只使用提供来的公钥进行加密，当服务器解锁时，客户端会使用提供来的私钥进行解密。使用的公私钥都是一次性，即客户端完成公钥和私钥的索取后，该公私钥自动失效，后续相同客户端触发锁定和解锁功能时，会再次生成一个公私钥。
  7.3、客户端和服务端的通信，都使用会话加密技术，防止数据泄漏。
  7.4、客户端功能均为模块化组合，可以在服务器端配置模块，以启用对应功能。同时模块化组合是为了让客户端功能可随时扩展。
  7.5、客户端可获取服务器策略，用以指定是否开启某些模块功能。
8、根据客户端功能，需要完成一个服务器，使用python语言开发，命名为WindSentinelAdmin，在创建服务器时，使用python虚拟环境，服务器功能定义为：
  8.1、接收客户端健康状态上报并展示
  8.2、接收客户端上传日志，并解压和解密传送来的日志，然后使用服务器的规则进行分析，该规则符合yarn编写规范，内置一些已知的mac恶意行为/hash/网络地址规则，同样也支持自定义规则。当命中规则时，会自动下发策略给客户端，让客户端杀死异常进程并断开该进程进程树关联进程的所有网络连接。
  8.3、对客户端进行策略下发和控制。策略包括指定进程断开网络，完全断开网络（除了agent能够和服务器网络连接外，其他任何进程都不允许使用网络），使用shell控制，锁定/解锁
  8.4、服务器可生成客户端，生成的客户端自动包含所有功能，但是针对所有功能，可以在服务端下发策略时配置是否开启。
  8.5、服务器对客户端管理，可以单个管理，也可以分组管理。
  8.6、服务器登录时，需要绑定MFA设备码
  8.7、服务器默认管理员为windmap,密码为admin，但是windmap用户登录后，必须第一时间绑定mfa并修改密码，才能正常使用功能，密码需要符合强密码复杂度，修改密码时，必须输入mfa和原始密码进行验证。
  8.8、服务器还有用户管理功能，日志查看功能。用户管理包括管理员用户与权限、审计员用户与权限。日志查看功能包括系统记录服务器的用户活动日志，同时还包含对应客户端上报的日志，可以使用sql语句进行针对性查询。
